1. 프로젝트가 뭔가

수리논술 첨삭 사업의 정적 랜딩 페이지. 차별화 포인트 두 가지:

  1. 학교별 기출 DB + 출제 경향 — 검색 유입("연세대 수리논술 기출" 류)이 목표. 차별화는 원장 첨삭 데이터 기반 "등급대별 완답률" 표기(경쟁자가 못 베끼는 지표).
  2. 수시 6장 조합 시뮬레이터 — 시험일 겹침 체크, 오전/오후 분산·이동 동선, 유형 유사도(준비 시너지), 4개 이상 선택 시 남은 카드 추천. URL 해시(#s=yonsei,korea,...)로 조합 공유 → 커뮤니티 바이럴 목표.

2. 파일 구조

파일역할
index.html페이지 뼈대 (히어로 → 시뮬레이터 → 기출 DB → 푸터)
style.css스타일 (모바일 반응형 완료)
data.js⭐ 단일 데이터 소스. 대학·시험일·최저·유사도·기출 전부 여기. 데이터 갱신 = 이 파일만 수정
app.js시뮬레이터·DB 렌더링 로직. 데이터 갱신 시 건드릴 필요 없음
PROJECT_STATUS.html이 문서 (내부용 — 배포 시 제외할 것)

프레임워크 없음. index.html 더블클릭으로 로컬 확인, 아무 정적 호스팅에나 폴더째 업로드하면 배포 끝.
